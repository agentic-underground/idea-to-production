**Brief**: Per-template parameter UIs / presets for common figure kinds (hero, diagram-accent, texture),
so a caller picks a preset instead of tuning raw params.

## User stories
- AS an illustrator I WANT presets for common figure kinds SO THAT I get a good result without learning
  each template's parameter space.

## EARS specification
- `EARS-001` (Ubiquitous): THE SYSTEM SHALL offer per-template presets/parameter UIs for common figure
  kinds over the allowlisted templates.
- `EARS-002` (Event-driven): WHEN a preset is selected THE SYSTEM SHALL apply its validated parameter set
  to the template's allowlisted schema.

## Acceptance criteria
1. Given a "hero" preset, When selected, Then its parameters are applied within the template's validation
   bounds and a figure is produced.

## Human interface test plan
None — presets are validated parameter sets over the MCP surface. Proof: selecting the "hero" preset
applies its parameters within the template's bounds and yields a figure (asserted against a stubbed
ComfyUI); a preset that would exceed bounds is rejected by the Phase-1 validators.

## Implementation notes
- Builds on the Phase-1 allowlist/validation core (PLAN_0047); **presets are validated parameter sets, not
  new node graphs** — they cannot bypass enforcement.
- ComfyUI stubbed in tests (deterministic).
