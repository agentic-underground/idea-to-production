# Add the publish multi-format export matrix (docx / epub / html via pandoc)

**Brief**: Extend `/publish:publish` with a `format=` matrix beyond markdown/pdf — `docx`, `epub`, and
single-file styled `html` via a pandoc-backed renderer — so one publish source reaches every channel from
one source. (The plugin already ships `handler-docx` for Word; this generalises the export surface and
establishes the shared `format=` dispatch the rest of EPIC_0018 extends.)

## User stories
- AS a publisher I WANT to export one article as docx, epub, or single-file HTML SO THAT I reach every
  channel without re-authoring per format.

## EARS specification
- `EARS-001` (Ubiquitous): THE SYSTEM SHALL render a publish article to `docx`, `epub`, and single-file
  styled `html` from the same source, selected by a `format=` argument.
- `EARS-002` (Event-driven): WHEN a supported `format=` is requested THE SYSTEM SHALL invoke the
  pandoc-backed renderer with the matching template and embed figures/assets inline where the format
  requires it (e.g. single-file html).
- `EARS-003` (Unwanted): IF pandoc (or a required format engine) is absent THEN THE SYSTEM SHALL report
  the missing prerequisite and which formats remain available, and SHALL NEVER emit a corrupt or empty
  file.

## Acceptance criteria
1. Given an article, When `format=docx|epub|html` is requested, Then a valid file of that type is produced
   from the one source.
2. Given a single-file `html` request, When rendered, Then images/styles are inlined so the file is
   self-contained.

## Human interface test plan
None — `/publish:publish format=…` is a CLI/skill invocation, not a UI element. Proof is that each format
produces a valid, openable file (validated structurally) and the html is self-contained.

## Implementation notes
- Extend `/publish:publish` with the `format=` matrix and the shared **dispatch** that routes to a
  renderer — this dispatch is EPIC_0018's shared infrastructure (slides/audio-script extend it).
- Route the new formats to a **pandoc** renderer; reuse `handler-docx`'s reference-doc/accessibility
  discipline for the docx path.
- Declare pandoc + per-format engines in the publish `check` manifest
  (`plugins/publish/skills/check/requirements.tsv`) — and keep the probes **no-download**
  (`command -v …`), so `scripts/verify-prereqs.sh` checks B and G stay GREEN.
