**Brief**: A `slide-deck` skill that generates presentation decks from an article or outline — Marp
markdown → HTML/PDF (or Beamer for LaTeX users). It reuses `diagram-studio` for figures sized to 16:9 and
the writer's narrative discipline (one idea per slide, a hook on each section), surfaced as a
`format=slides` target on `/publish:publish`.

## User stories
- AS a builder I WANT to turn a finished article into a slide deck SO THAT I can present the same material
  without re-authoring it.
- AS a presenter I WANT figures sized for 16:9 and one idea per slide SO THAT the deck reads cleanly on a
  projector.

## EARS specification
- `EARS-001` (Ubiquitous): THE SYSTEM SHALL produce a presentation deck from an article or outline as
  another `/publish:publish` format (`format=slides`).
- `EARS-002` (Event-driven): WHEN `format=slides` is requested THE SYSTEM SHALL render Marp markdown to
  HTML/PDF (or Beamer when the project targets LaTeX), embedding `diagram-studio` figures sized to 16:9.
- `EARS-003` (Optional): WHERE a Beamer/LaTeX toolchain is configured THE SYSTEM SHALL offer Beamer
  output in addition to Marp.

## Acceptance criteria
1. Given an article, When `/publish:publish format=slides` runs, Then a deck (HTML or PDF) is produced with
   one primary idea per slide and a hook on each section.
2. Given a section that references a figure, When the deck is built, Then the figure is rendered via
   `diagram-studio` at a 16:9-appropriate size.

## Human interface test plan
None — `format=slides` is a skill invocation. Proof: the produced deck opens, paginates one-idea-per-slide,
and embeds 16:9 figures (validated structurally on the rendered output).
