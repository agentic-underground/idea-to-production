**Brief**: When a developer installs the idea-to-production marketplace on a new machine and opens their
first session, the harness asks once â€” *"Set up GitHub MCP server now? [yes/no/dontask]"*. `yes` runs a
bundled, idempotent installer that wires the `github` MCP server into `~/.claude/.mcp.json`; `no` skips
this session; `dontask` persists a flag so the prompt never returns.

## User stories
- AS A marketplace developer installing i2p for the first time I WANT a guided one-step prompt to wire up
  the GitHub MCP server SO THAT I can use GitHub-aware agents without finding and editing config by hand.
- AS A developer who knows I don't want the GitHub MCP I WANT to answer "dontask" once SO THAT the prompt
  disappears permanently.
- AS A developer whose machine lacks Docker I WANT the script to detect the missing dependency and offer
  an npx-based fallback or a plain explanation SO THAT I still get a clear path.

## EARS specification
- `EARS-001` (Event-driven): WHEN a new i2p session starts AND `~/.claude/.mcp.json` does NOT already
  contain a `github` MCP entry AND the `dontask` flag file is absent THE SYSTEM SHALL invoke the GitHub
  MCP setup prompt (`scripts/setup-github-mcp.sh --prompt`).
- `EARS-002` (Event-driven): WHEN the user answers `yes` THE SYSTEM SHALL run the installer â€” detect
  Docker, collect or reuse a GitHub PAT, write the `github` entry to `~/.claude/.mcp.json`, and print a
  confirmation.
- `EARS-003` (Event-driven): WHEN the user answers `no` THE SYSTEM SHALL exit the prompt without
  modifying any file and continue the session.
- `EARS-004` (Event-driven): WHEN the user answers `dontask` THE SYSTEM SHALL write
  `~/.claude/hook-state/github-mcp-dontask` and exit without modifying `.mcp.json`.
- `EARS-005` (Unwanted): IF `~/.claude/.mcp.json` already contains a valid `github` MCP entry THEN THE
  SYSTEM SHALL NOT display the prompt and SHALL return exit code 0 immediately (idempotent).
- `EARS-006` (Unwanted): IF Docker is absent AND npx is absent THEN THE SYSTEM SHALL print an install
  guide for both and exit without modifying any file (detect-and-guide, never halt).
- `EARS-007` (Unwanted): IF the provided GitHub PAT is empty or whitespace-only THEN THE SYSTEM SHALL
  reject it and re-prompt (or abort on a second failure).
- `EARS-008` (State-driven): WHILE `~/.claude/hook-state/github-mcp-dontask` exists THE SYSTEM SHALL NOT
  invoke the setup prompt in any subsequent session.
- `EARS-009` (Optional): WHERE `GH_TOKEN` is already set in the environment THE SYSTEM SHALL offer to
  reuse it as the PAT rather than prompting for a new one.

## Acceptance criteria
1. On a clean install (no `.mcp.json`, no dontask flag), opening a session triggers the prompt.
2. Answering `yes` with a valid PAT and Docker available: `.mcp.json` is created/updated with the correct
   `github` entry; the next session start skips the prompt.
3. Answering `no`: no files are modified; the next session start prompts again.
4. Answering `dontask`: `~/.claude/hook-state/github-mcp-dontask` is created; the next session start does
   NOT prompt.
5. Re-running when `.mcp.json` already has the `github` entry: exits 0 silently with no file changes.
6. Docker absent, npx absent: a human-readable guide is printed; no files modified; exit code 0.
7. `GH_TOKEN` present: the script offers to reuse it; the user can accept or supply a different token.

## Implementation notes
- **Script:** `scripts/setup-github-mcp.sh` (marketplace root, not plugin-specific).
- **Harness hook:** a SessionStart hook in the `i2p` front-door plugin
  (`plugins/i2p/hooks/setup-github-mcp-prompt.sh`) that delegates to the script only when conditions are
  met (no existing entry, no dontask flag). It must read only the state file and `~/.claude/.mcp.json` â€”
  never assume other plugins are installed.
- **`.mcp.json` target:** `~/.claude/.mcp.json`. **Merge, never overwrite** â€” preserve other MCP entries
  (ansible, playwright, â€¦).
- **Docker image:** `ghcr.io/github/github-mcp-server` (verified working). **npx fallback:**
  `npx @modelcontextprotocol/server-github`.
- **State file:** `~/.claude/hook-state/github-mcp-dontask` (follows the existing hook-state convention,
  e.g. the welcome-declined flag). Never write hook-state into a user repo.
- The new hook must pass `scripts/verify-prereqs.sh` check L (hooks smoke-exec: exists, `bash -n`, exits
  0 on a synthetic SessionStart event in a sandbox).
