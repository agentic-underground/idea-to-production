## Shared infrastructure
- **The lifecycle test surface** — the `lifecycle-orchestrator` agent (`plugins/foundry/agents/`) and its
  sentinel/handoff chain are the system under test for the retrofit. PLAN_0011 (the only current child)
  authors the spec + proof against this surface; any future child plan in this EPIC reuses the same
  fixture-roadmap-item harness rather than rebuilding it.
- **Provenance caveat (load-bearing for the builder).** The legacy umbrella (item 32) named two children:
  an interactive *"Merge PR now?"* confirmation (legacy item 33) and the PR #56 quality retrofit (legacy
  item 34). Under the FLEET v2 pipeline the **engine owns land + merge** (the `delivery: auto|direct|pr`
  contract — see `plugins/foundry/skills/roadmapper/references/fleet-pipeline-standard.md`), so the
  interactive-merge child is **superseded** by engine-governed delivery and is not re-emitted as a PLAN.
  Item 34 (the quality retrofit) carries forward as PLAN_0011. Legacy item 33 has no backlog file in this
  batch; it is recorded here as superseded, not silently dropped.

## Definition of done
- Every child PLAN of this EPIC is `completed` (the engine rolls EPIC completion up from its PLANs).
- `.pipeline/verify` is GREEN for each landed PLAN; the conformance gate (`scripts/verify-prereqs.sh`
  check P) stays GREEN over `docs/roadmap/`.

## Execution contract (for the builder)
The engine builds one PLAN at a time in `## Plans` order, running each PLAN's `## Construction process`
(which invokes FOUNDRY scoped to the `(EPIC, PLAN)` pair). A failed PLAN leaves its manifest row
`engaged` and resumes next tick; the EPIC never lands until all its PLANs are GREEN.
