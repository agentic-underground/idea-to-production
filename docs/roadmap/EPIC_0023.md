## Shared infrastructure
- **The charting-matrix figure pipeline** — `diagram-studio`/`illustrator` handlers + the
  `design-reviewer` legibility checks. PLAN_0027 (diagram round-tripping) and PLAN_0028 (localisation)
  both re-render figures through this pipeline; PLAN_0028's **figure label-externalisation convention**
  (a labels map per figure) is the shared piece localisation needs and round-tripping should respect —
  whichever lands first owns the convention, the other consumes it.
- **The cross-format render path** — citations (PLAN_0024) and themes (PLAN_0026) apply *across* every
  output format (markdown/PDF/HTML/slides), so they hook the same render path EPIC_0018 extends rather
  than a single format. (No build-order dependency on EPIC_0018; these aids degrade to the formats present.)
- **Rename caveat (load-bearing).** Legacy items name the plugin **PRESSROOM** (`plugins/pressroom/`,
  `/pressroom:publish`); it is now **`publish`** (`plugins/publish/`, `/publish:publish`). Target the
  current `publish` plugin.

## Definition of done
- All five PLANs are `completed` (the engine rolls EPIC completion up from its PLANs).
- `.pipeline/verify` is GREEN for each landed PLAN; `scripts/verify-prereqs.sh` (incl. check P) stays

## Execution contract (for the builder)
The engine builds one PLAN at a time in `## Plans` order, running each PLAN's `## Construction process`
(FOUNDRY scoped to the `(EPIC, PLAN)` pair). A failed PLAN leaves its row `engaged` and resumes next tick;
the EPIC lands only when all five PLANs are GREEN.
