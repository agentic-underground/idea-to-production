## Construction process
1. Inventory the deterministic, offline checks this repo can support: all plugin/marketplace manifests
   parse and conform to their schema; `tests/` (if runnable) pass; tracked Python compiles.
2. Add a minimal runner for the existing `tests/` if one is missing (a `pyproject.toml`/runner as fits),
   and a small manifest-schema check.
3. Upgrade `.pipeline/verify` from the seed to that real gate (deterministic + offline + fast). Keep it
   green.
4. **Do NOT modify the marketplace plugins or the flow-server/MCP surface** — only add CI scaffolding.

## Definition of done
- `.pipeline/verify` runs a real gate (manifests + tests) and is green.
- No change to plugin behavior or the flow-server/MCP surface.

## Execution contract (for the builder)
Branch `pipeline/0000-ci-gate` → add CI → gate green → propose a PR. Do not edit
`docs/roadmap/.pipeline.md`. A failed tick leaves the row `engaged` to resume.
