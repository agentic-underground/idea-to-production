**Brief**: `/code-quality --diff HEAD~1` analyses only changed lines; output formatted as GitHub PR review
comments; integration with `gh pr review`.

## User stories
- AS a reviewer I WANT code-quality to analyse only the diff and post its findings as PR review comments SO
  THAT feedback lands inline on the changed lines.

## EARS specification
- `EARS-001` (Ubiquitous): THE SYSTEM SHALL support analysing only the lines changed since a ref and
  emitting findings as GitHub PR review comments.
- `EARS-002` (Event-driven): WHEN `/code-quality --diff <ref>` runs THE SYSTEM SHALL restrict analysis to
  the changed lines and format output for `gh pr review`.
- `EARS-003` (Unwanted): IF a finding's context spans unchanged lines THEN THE SYSTEM SHALL note that
  diff-mode is line-scoped (not a full-file review).

## Acceptance criteria
1. Given a diff, When `--diff HEAD~1` runs, Then only changed lines are analysed.
2. Given the findings, When posted, Then they are valid `gh pr review` comments on the right lines.
