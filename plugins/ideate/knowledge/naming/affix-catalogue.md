# Affix catalogue — prefixes & suffixes as hooks

The "extra catchiness / a hook" layer. A productive affix bolts onto a root to (a) make a taken root
ownable, (b) signal a category, and (c) give a name rhythm. Applied as a **transform across veins**:
take a vein's roots, attach charter-appropriate affixes, then re-check availability. Each affix
*signals* — choose ones that match the archetype, don't just decorate.

## Suffixes

| Suffix | Signal | Examples |
|---|---|---|
| **-ify / -ize** | transformation, agency, "make into" | Spotify, Shopify, Notify |
| **-ly** | friendly, approachable, adverbial (SaaS staple) | Bitly, Grammarly, Calendly |
| **-o / -io / -a** | warm, playful, Romance flavour; data-tool trend | Twilio, Grafana, Kibana |
| **-us / -um / -is** | Latinate gravitas, science, tradition | Nexus, Opus, Basis |
| **-or** | agentive — "the one who does" | Provenor, Creator, Advisor |
| **-ia** | place/abstraction, elegance | Nokia, Wikipedia, Serenia |
| **-hub** | ecosystem, central gathering place | GitHub, StubHub, HubSpot |
| **-base** | foundation, infrastructure, reliability | Firebase, Coinbase, Airtable |
| **-labs** | R&D, experimentation, cutting-edge | (Google) Labs, Sketch Labs |
| **-stack** | completeness, layered system | Fullstack, Substack |
| **-kit** | ready-to-use toolkit, accessibility | Stripe Kit, RevenueKit |
| **-ex / -ix / -yx** | modern, abstract, tech-energy | Xerox, Netflix, Onyx |
| **-scape** | scope, environment, expansiveness | Landscape, Cloudscape |
| **-ster** | youthful, informal, an identity | Napster, Friendster |
| **-gram** | message, record, sharing | Instagram, Telegram |
| **-ai** | AI-native positioning (2020s) | (many AI startups) |

## Prefixes

| Prefix | Signal | Examples | Note |
|---|---|---|---|
| **e-** | electronic, online | eBay, email | **dated** — avoid for freshness |
| **i-** | innovation/identity (Apple-coded) | iPhone, iPad | 2000s-coded |
| **get- / try- / use-** | accessibility, imperative, domain-friendly | GetVerified, TryX, Usenet | great when the bare word is taken |
| **go-** | forward motion, ease | Go (lang), GoPro | ties to a "launch/we-are-go" SOUL |
| **neo-** | new/modern, disruption | Neobank | |
| **meta-** | beyond, self-referential, abstract | Meta, Metadata | |
| **omni- / uni-** | all-encompassing / unity, simplicity | Omnicare, Uniqlo | |

## 2020s trends

- **-ly / -ify** still dominate SaaS; now slightly expected — use with a distinctive root.
- **-ai** suffix and **AI-native** roots surging; risks dating the name to the era.
- **Hybrid neoclassical** (Greek/Latin prefix + English noun: *GraphQL, AutoML*) signals "technical +
  accessible."
- **get-/try-** prefixes thrive because they unlock taken roots **and** a clean domain.

## Discipline

- An affix must earn its place — it should add a *signal*, not just availability. A mismatched affix
  (a playful *-o* on a Ruler-archetype security brand) reads wrong.
- Re-run the affixed form through the checker: the affix often *creates* the clean namespace (e.g.
  `provenor`, `quasgo`) — but can also create a neighbour cluster, so verify.
- Don't stack affixes (`get-X-ify`) — one hook per name.
