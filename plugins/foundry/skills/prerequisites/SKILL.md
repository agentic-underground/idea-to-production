# FOUNDRY — Generate PREREQUISITES.md

Produces a single, project-local `PREREQUISITES.md` that tells a developer (or a provisioning agent)
exactly what software the installed plugins want, why, how to install it, and what is currently
missing on this machine.

## How to run

1. **Detect which plugins are installed** in this marketplace/session — foundry is always present;
   include sentinel and/or pressroom if their plugin directories exist alongside foundry.

2. **Run each installed plugin's check** to capture live status (advisory):
   ```bash
   bash ${CLAUDE_PLUGIN_ROOT}/skills/check/scripts/check.sh
   # if sentinel present:  bash <sentinel>/skills/check/scripts/check.sh
   # if pressroom present: bash <pressroom>/skills/check/scripts/check.sh
   ```

3. **Assemble `PREREQUISITES.md`** in the **current project root** with these sections, drawing the
   canonical prose from the marketplace [`PREREQUISITES/`](../../../../PREREQUISITES/) folder
   (`00-core.md`, `10-foundry.md`, and `20-sentinel.md`/`30-pressroom.md` for whichever companions
   are installed, plus `40-mcp.md` and `45-lsp.md`):

   ```markdown
   # PREREQUISITES — <project name>
   > Generated by /foundry:prerequisites on <date>. Plugins detected: <list>.

   ## Core baseline            (from 00-core.md)
   ## FOUNDRY                  (from 10-foundry.md — toolchains, Playwright MCP, debuggers, LSP)
   ## SENTINEL  (if installed) (from 20-sentinel.md — scanners + Semgrep MCP)
   ## PRESSROOM (if installed) (from 30-pressroom.md — dual-engine typesetting + DTP)
   ## MCP servers              (from 40-mcp.md)
   ## Language servers         (from 45-lsp.md)

   ## Current machine status
   <paste the ✓/✗ tables from step 2, so the reader sees exactly what is missing here>

   ## Install
   <the relevant install one-liners; point at PREREQUISITES/ansible/ for provisioning>
   ```

4. Keep it **scoped** — only document plugins that are actually installed. Do not invent tools; the
   authoritative lists are the per-plugin `skills/check/requirements.tsv` files and the
   `PREREQUISITES/` folder.

## Relationship to `/foundry:check`

`-check` answers "is it installed *right now*?" (a probe). `-prerequisites` answers "what should be
installed, why, and how?" (a document) — and embeds the check output as the live status snapshot.
