# Density & Cognitive Load

## Density: moderate default, customer-toggleable
Some customers love dense (more on screen, fewer scrolls); some need spacious (calm, clear). Set the **median = moderate** and offer a **spacious ↔ dense toggle**. Phones lean spacious/tap-light; large-screen keyboard users tolerate dense.

## Cognitive load budgets
Lean on established theory, applied per screen and per panel:
- **Cognitive Load Theory (Sweller):** minimise *extraneous* load (clutter, inconsistency); protect capacity for the *intrinsic* task.
- **Hick's Law:** decision time grows with the log of the number of choices — constrain and group options (this is *why* lookups beat free text and why menus have depth).
- **Miller's 7±2:** keep a group/panel within working-memory reach; chunk beyond it.

### Practical budgets (defaults, present-then-question)
- One **primary** action or stat per panel; everything else visibly secondary.
- ≤~5 inputs per group in words-style data entry before chunking into groups/steps.
- A conditional-field-group exists precisely to keep only the *relevant* fields in view.

## Tie-ins
Density interacts with `three-modalities.md` (touch → lighter) and `layout-and-flow.md` (grouping, balance). Always keep accessibility constant across densities.
