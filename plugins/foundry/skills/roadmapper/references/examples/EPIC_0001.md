**Value proposition**: A person with things to do needs a single trustworthy place to capture them
and see what is still outstanding. TaskFlow's core is the smallest such place: add a task, mark it
done, see the list survive a reload. Everything else (due dates, tags, sharing) is later EPICs built
on this spine.

**Brief**: The foundational capability of TaskFlow — a durable task list a single user can add to,
complete items in, and read back. Decomposed into three dependency-ordered vertical slices, the
first of which (`PLAN_0001`) also lifts the shared persistence layer the later slices reuse.

## Shared infrastructure
- **Task store** (the persistence layer — a JSON-file-backed repository with `add` / `update` / `list`).
  **Owned by `PLAN_0001`**: the first slice creates it as the thinnest thing that lets a task be added
  and read back. `PLAN_0002` (toggle complete) and `PLAN_0003` (list filtering) reuse it and MUST NOT
  rebuild it.
- **`Task` model** (`id`, `title`, `done`). Also created in `PLAN_0001`; the `done` field is written by
  `PLAN_0001` (defaulting `false`) and toggled by `PLAN_0002`.
