# Look-and-Feel — commit to a direction

Borrowed from the frontend-design discipline: pick **one** clear aesthetic direction and execute it precisely. Intentionality beats intensity; both refined-minimal and bold-maximal work when committed to.

## Tone vocabulary (pick one, then elaborate)
- **Refined / luxury** — restraint, precise spacing, quiet confidence.
- **Utilitarian / industrial** — function-forward, dense-friendly, no ornament.
- **Editorial / magazine** — strong typographic hierarchy, generous rhythm.
- **Playful / toy-like** — rounded, characterful, motion-friendly.
- **Brutalist / raw** — exposed structure, high contrast, grid-honest.
- **Organic / natural** — soft transitions, warm palette.
- **Retro-futuristic / technical** — instrument-panel energy (suits operation-style).

## Rules
- **Dark mode is the default**; design the light theme as a deliberate override, not an afterthought.
- Use **CSS custom properties** as tokens (colour, type, spacing, radius, shadow, motion) so the direction is consistent and themeable.
- Avoid generic defaults; let the *customer* and *paradigm* shape the choice (a reader's library and a back-office grid should not feel identical).
- Reduced-motion and contrast obligations hold regardless of the aesthetic chosen.

Commit to tokens early; record the direction so later agents extend it rather than dilute it.
