# Phonosemantics — making the sound carry the meaning

Sound *means*, before semantics. A name's phonetic texture should reinforce the charter's archetype
and values — independent of any literal meaning. This is the layer Lexicon Branding used to name
Pentium, Swiffer, Febreze, BlackBerry, Dasani.

## The core effects

- **Bouba/kiki.** Rounded, sonorant sounds (b, m, l, o, u — *bouba*) read **soft, smooth, friendly,
  large**. Sharp, plosive sounds (k, t, p, x, i — *kiki*) read **fast, precise, edgy, small**. Pick
  the family that matches the personality. (Robust cross-cultural effect.)
- **Plosives vs sonorants.** Plosive **initials** (b, p, t, d, k, g) are *punchy, memorable,
  distinctive* — disproportionately common in strong brands (Google, Kodak, Pepsi, GoPro). Sonorants
  (l, m, n, r) feel *calm, trustworthy, fluid* (Lululemon, Allianz, Aramco).
- **The "K".** The hard *k* sound reads as **sharp, energetic, and faintly comedic** — long a
  copywriter's tool (Kodak, Kleenex, Kit-Kat, Coca-Cola). Use for energy/wit; avoid for solemnity.
- **Front vs back vowels.** Front vowels (ee, i, e — *Mini, Kindle, Twitter*) connote **small, fast,
  light, sharp**. Back vowels (oo, oh, ah — *Google, Volvo, Honda*) connote **large, heavy, solid,
  serious**. Match vowel to the size/weight the brand wants to feel.
- **Voiced vs voiceless.** Voiced consonants (b, d, g, v, z) feel heavier/rounder; voiceless (p, t, k,
  f, s) feel lighter/faster.

## Rhythm, meter, length

- **Brevity.** ≤4 syllables (sweet spot 1–2). Longer names get abbreviated by users (Federal Express →
  FedEx). See Neumeier criterion 2 in [`frameworks.md`](frameworks.md).
- **Trochaic preference.** Stress-first (DA-da: *Google, Stripe-ly, Vercel*) scans more naturally in
  English than iambic.
- **CVC-CVC / reduplication** (TikTok, Kit-Kat) maximises memorability via rhythm.
- **Alliteration & assonance** (Coca-Cola, PayPal, Dunkin' Donuts; Google's oo-oo) boost recall and
  "mouth-feel."
- **Avoid hostile clusters.** Word-initial *str-, scr-, sphr-* and tongue-twisters tank
  pronounceability across non-native speakers (a SCRATCH "hard-to-pronounce" flag). The TQXR lesson:
  an all-consonant cluster is unsayable — it can be a handle, never a spoken brand.

## Archetype → sound map

Pair with the chosen Jung archetype ([`frameworks.md`](frameworks.md)):

| Archetype | Sound palette |
|---|---|
| **Hero / Ruler** | hard initials (k, g, t), back vowels, strong stress — *power & weight* |
| **Magician / Creator** | coined neologisms, unexpected phoneme combos, mid-word x/z — *mystery* |
| **Explorer / Outlaw** | plosives, imperative cadence, abrupt endings — *motion & edge* |
| **Sage / Ruler** | Latin/Greek roots, measured polysyllables — *authority* |
| **Lover / Caregiver** | soft m/n/l, warm round vowels, flowing rhythm — *intimacy* |
| **Jester / Everyman** | the comedic "k", reduplication, diminutives, short words — *play & warmth* |
| **Innocent** | soft consonants, *-y/-ie* diminutives, bright front vowels — *gentleness* |

## How to apply

In generation, after a vein produces candidates, run a **phonosemantic pass**: keep the ones whose
sound *already* matches the archetype palette; nudge the rest (swap a sonorant initial for a plosive,
a back vowel for a front one). In evaluation, **sound-symbolism-fit** is a scored axis. Never let a
literally-apt name survive if it *sounds* wrong for the archetype.
