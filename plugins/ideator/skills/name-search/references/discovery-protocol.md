# Discovery protocol — the naming brief interview

Naming begins with a brief, not a guess. This is the **discover** phase: reach knowledge-parity on
*who* the name is for and *what* it must do, then feed the enriched charter
([`charter-protocol.md`](charter-protocol.md)). It mirrors the house dialogue discipline used by
`market-scanner`'s `/goal` and `ideator`'s `ideate` challenge protocol.

## The dialogue discipline (non-negotiable)

**Infer-first, then ask the gaps.** Pull everything you can from any upstream IDEA package, the
`ideate` output, or a bounded local trawl (see charter-protocol). Then ask **only the load-bearing
unknowns** — the ones that would change which names you generate. Discipline:

- **One focused question per turn**, each with a **recommended answer + multiple-choice** options.
- A left-blank / "surprise me" answer is valid — it *widens* the net for that axis.
- Never a wall of questions; never silent enforcement — present the default, name the trade-off, let
  the user decide.
- Every resolved ambiguity is **written into the charter** so it is never re-asked (waste-elimination).
- Stop when the charter's load-bearing fields are filled. Over-interrogation is its own failure.

## The brief field-set

Infer each first; ask only where the answer is undetermined **and** load-bearing.

1. **Product & problem** — what it is, the pain it kills, the benefit (one layperson sentence).
2. **Audience** — who *specifically* (a named role, not "users"); demographics + psychographics; and
   *how they speak* (terse/dev, warm/consumer, formal/enterprise). Drives register.
3. **Brand archetype** — pick **1–2** of Jung's 12 ([`../../../knowledge/naming/frameworks.md`](../../../knowledge/naming/frameworks.md)).
   Drives the sound palette ([`../../../knowledge/naming/phonosemantics.md`](../../../knowledge/naming/phonosemantics.md))
   and favoured name-types.
4. **Positioning vs the competitive set** — 5–10 rival names and what *approach* they use (descriptive/
   suggestive/abstract); name the gap to occupy so we don't sound like them.
5. **Ranked values + emotional goal** — what the name must encode, in priority order; and the *feeling*
   it should trigger on first encounter.
6. **Name-type appetite** — descriptive ↔ suggestive ↔ abstract; real-word ↔ compound ↔ coined.
   **This selects which veins fire** ([`../../../knowledge/naming/name-types.md`](../../../knowledge/naming/name-types.md))
   — the single biggest lever on what the search returns.
7. **Power-adjacency targets** — concepts/myths/sciences to borrow equity from, if any
   ([`../../../knowledge/naming/dev-tool-conventions.md`](../../../knowledge/naming/dev-tool-conventions.md)).
8. **Intellectual-humour appetite** — none ↔ subtle insider wink ↔ overt pun.
9. **Risk appetite** — safe ↔ bold/provocative; tolerance for coined vs real-word.
10. **Extendability** — must it stretch to future product lines / an org of products? (kills
    over-narrow names; a Neumeier/SCRATCH axis).
11. **Hard constraints** — must-include / must-avoid words; syllable target/range; banned/saturated
    stems; languages where connotation matters; exemplar brands the user admires.

## Output

A filled brief that becomes the **enriched charter**. The charter's name-type strategy + archetype +
risk appetite directly parameterise generation ([`generation.md`](generation.md)) and the scored
challenge ([`evaluation-rubric.md`](evaluation-rubric.md)). If a load-bearing field can't be resolved,
say so and record it as an explicitly accepted assumption — never paper over it.
