# Add the unified .securityignore (one exclusion file, per-lens sections)

**Brief**: One exclusion file consumed by all security lenses (superseding the per-skill
`.piiignore` / `.secretignore`), with per-lens sections plus a shared/global section, so exclusions live
in one auditable place. This is shared infrastructure the other lenses read — it lands first.

## User stories
- AS a security owner I WANT one `.securityignore` for every lens SO THAT exclusions are auditable in a
  single file rather than scattered per-skill.

## EARS specification
- `EARS-001` (Ubiquitous): THE SYSTEM SHALL read a single `.securityignore` (with per-lens sections) as
  the exclusion source for every security lens.
- `EARS-002` (Event-driven): WHEN a lens runs THE SYSTEM SHALL apply that lens's section of
  `.securityignore` plus any shared/global section.
- `EARS-003` (State-driven): WHILE a legacy `.piiignore`/`.secretignore` exists THE SYSTEM SHALL honour it
  for backward compatibility and surface a deprecation note pointing at `.securityignore`.

## Acceptance criteria
1. Given a `.securityignore` with a secrets section, When the secret-scan runs, Then those patterns are
   excluded and no other lens's exclusions leak into it.
2. Given a legacy `.secretignore`, When the secret-scan runs, Then it is still honoured with a deprecation
   note.

## Human interface test plan
None — `.securityignore` is a config file read by the lenses. Proof: a fixture with a secrets section
excludes those patterns from `scan-for-secrets` and nothing else; a legacy `.secretignore` is still
honoured with a deprecation note (asserted on lens output).

## Implementation notes
- Define the `.securityignore` format (per-lens sections + a shared section).
- Update each lens (`scan-for-pii`, `scan-for-secrets`, `scan-dependencies`) to read it; keep the legacy
  `.piiignore`/`.secretignore` fallbacks with a deprecation note.
- **Rename caveat:** target the current `security` plugin and its `scan-*` verbs (legacy text says
  SENTINEL / `pii-audit` / `secret-scan` / `dependency-audit`).
