# Add container & IaC scanning lenses (Dockerfiles, Terraform, k8s)

**Brief**: Extend the security lenses to `Dockerfile`s (root user, unpinned base images, secrets in
layers) and IaC (`*.tf`, k8s manifests) for misconfigurations and embedded secrets — so infra risks are
caught by the same gate as code risks.

## User stories
- AS an operator I WANT the security gate to scan my Dockerfiles and IaC for misconfigurations and
  embedded secrets SO THAT infra risks are caught by the same gate as code risks.

## EARS specification
- `EARS-001` (Ubiquitous): THE SYSTEM SHALL scan `Dockerfile`s and IaC sources (`*.tf`, k8s manifests) for
  misconfigurations and embedded secrets as additional security lenses.
- `EARS-002` (Event-driven): WHEN a `Dockerfile` is scanned THE SYSTEM SHALL flag a root user, an unpinned
  base image, and secrets in layers; WHEN IaC is scanned THE SYSTEM SHALL flag misconfigurations and
  embedded secrets.
- `EARS-003` (Unwanted): IF a manifest format is unrecognised THEN THE SYSTEM SHALL skip it with a note,
  and SHALL NEVER silently report a clean pass over unscanned infra.

## Acceptance criteria
1. Given a Dockerfile running as root with an unpinned base, When scanned, Then both are reported.
2. Given a Terraform file with an embedded secret, When scanned, Then it is flagged.

## Human interface test plan
None — the lenses are scan operations. Proof: fixture Dockerfile (root + unpinned base) reports both; a
`.tf` with an embedded secret is flagged; an unrecognised manifest is skipped with a note (asserted on
output).
