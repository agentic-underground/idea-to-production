## Shared infrastructure
- **The `format=` dispatch surface on `/publish:publish`** — the single argument-routed entry that selects
  a renderer. PLAN_0019 (multi-format export) **establishes** this dispatch + the pandoc-backed renderer;
  PLAN_0020 (slides) and PLAN_0021 (audio-script) **extend** it with new `format=` values rather than
  rebuilding the routing. Lift the dispatch into PLAN_0019 so later PLANs never re-create it.
- **The figure/render pipeline** — `diagram-studio` figures and the writer's narrative discipline are
  reused across slides and exports; the per-format engine availability is declared in the publish
  `check` manifest (`requirements.tsv`).
- **Rename caveat (load-bearing for the builder).** The legacy items name the plugin **PRESSROOM** with
  paths `plugins/pressroom/` and `/pressroom:publish`. The plugin is now **`publish`** (`plugins/publish/`,
  `/publish:publish`). Target the current `publish` plugin; treat every "PRESSROOM"/"pressroom" reference
  in the migrated PLANs as "publish".

## Definition of done
- All four PLANs are `completed` (the engine rolls EPIC completion up from its PLANs).
- `.pipeline/verify` is GREEN for each landed PLAN; `scripts/verify-prereqs.sh` (incl. check P and the
  publish `check` manifest's well-formedness, B) stays GREEN.

## Execution contract (for the builder)
The engine builds one PLAN at a time in `## Plans` order, running each PLAN's `## Construction process`
(FOUNDRY scoped to the `(EPIC, PLAN)` pair). PLAN_0019 (the dispatch foundation) lands first. A failed
PLAN leaves its row `engaged` and resumes next tick; the EPIC lands only when all four PLANs are GREEN.
