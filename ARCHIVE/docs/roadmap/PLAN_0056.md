# Add the code-quality pre-commit hook

**Brief**: A pre-commit hook that runs the quality analysis on changed files only, with a configurable
threshold (fail the commit below score X) and a fast mode (lint only, no LLM call).

## User stories
- AS a developer I WANT a pre-commit hook that quality-checks only my changed files SO THAT regressions are
  caught before commit without scanning the whole tree.

## EARS specification
- `EARS-001` (Ubiquitous): THE SYSTEM SHALL provide a pre-commit hook that analyses changed files and can
  fail the commit below a configurable score threshold.
- `EARS-002` (Event-driven): WHEN the hook runs in fast mode THE SYSTEM SHALL lint only (no LLM call) for
  speed.
- `EARS-003` (Unwanted): IF the score is below the configured threshold THEN THE SYSTEM SHALL block the
  commit with the findings.

## Acceptance criteria
1. Given changed files and a threshold, When committing, Then the hook analyses only those files and blocks
   below threshold.
2. Given fast mode, When the hook runs, Then it lints without an LLM call.

## Human interface test plan
None — the hook is a git integration. Proof: against a fixture repo, committing changed files below
threshold blocks with findings; at/above passes; fast mode lints with no LLM call (asserted on the hook's
exit code + output, LLM call stubbed).

## Implementation notes
- Hook script scoped to the `git diff` changed-file set (the shared changed-scope helper — see EPIC_0054
  shared-infra, also used by PLAN_0058); configurable threshold; fast (lint-only) path that makes no LLM
  call.
- The hook must pass `scripts/verify-prereqs.sh` check L (exists, `bash -n`, exits 0 on a synthetic event
  in a sandbox) if shipped under a plugin's `hooks/`.
