**Brief**: A `coverage-gate-agent` that blocks merges below a coverage threshold, integrated into CI —
enforcing the coverage floor automatically.

## User stories
- AS a maintainer I WANT merges blocked when coverage falls below threshold SO THAT the coverage floor is
  enforced automatically in CI.

## EARS specification
- `EARS-001` (Ubiquitous): THE SYSTEM SHALL evaluate coverage against a configured threshold in CI and
  block a merge below it.
- `EARS-002` (Event-driven): WHEN coverage on a PR is below the threshold THE SYSTEM SHALL fail the gate
  and report the gap.

## Acceptance criteria
1. Given coverage below threshold, When the gate runs in CI, Then the merge is blocked with the shortfall.
2. Given coverage at/above threshold, When the gate runs, Then it passes.

## Human interface test plan
None — the gate is a CI agent over a coverage report. Proof: a fixture coverage report below threshold
fails the gate with the shortfall; at/above passes (asserted on the gate verdict; coverage input fixtured).

## Implementation notes
- Part of the v2.0 multi-agent pipeline (EPIC_0061); CI-integrated; configurable threshold. Complements the
  coverage-loop discipline (100% is the floor that results, not the target — see
  `plugins/foundry/skills/code-quality/`).
- Feeds the orchestrator (PLAN_0065); a below-threshold result is a blocking verdict.
