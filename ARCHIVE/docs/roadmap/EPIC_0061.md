## Shared infrastructure
- **The three quality agents are the shared verb surface the orchestrator composes.** PLAN_0062
  (spec-reviewer), PLAN_0063 (coverage-gate), and PLAN_0064 (architecture-guardian) each ship an
  independent agent; PLAN_0065 (orchestrator) fans them out per PR and synthesises one verdict
  (max-severity wins) — so the three agents land before the orchestrator (PLAN_0065 `depends_on` all
  three). The orchestrator mirrors the existing fan-out-then-synthesise pattern (e.g. `/foundry:pr-review`).
- **The `code-quality` engine + the coverage-loop discipline** (`plugins/foundry/skills/code-quality/`)
  underpin the agents — the coverage-gate agent builds on the 100%-is-the-floor coverage discipline; the
  architecture-guardian needs a way to declare boundaries (config/ADR, consistent with EPIC_0054's ADR
  integration).

## Definition of done
- All four PLANs are `completed` (the engine rolls EPIC completion up from its PLANs).
- `.pipeline/verify` is GREEN for each landed PLAN; `scripts/verify-prereqs.sh` (incl. check P, and check
  J four-mirror if a new agent ships its own skill surface) stays GREEN.

## Execution contract (for the builder)
The engine builds one PLAN at a time in `## Plans` order, running each PLAN's `## Construction process`
(FOUNDRY scoped to the `(EPIC, PLAN)` pair). The three agents (PLAN_0062/0063/0064) land before the
orchestrator (PLAN_0065). A failed PLAN leaves its row `engaged` and resumes next tick; the EPIC lands only
when all four PLANs are GREEN.
