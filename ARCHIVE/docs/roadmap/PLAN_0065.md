# Add the pipeline orchestrator (run all three agents per PR)

**Brief**: An orchestrator that runs the three quality agents (spec-reviewer, coverage-gate,
architecture-guardian) in parallel on every PR and synthesises one combined verdict (max-severity wins).

## User stories
- AS a team I WANT one orchestrator to run all three quality agents in parallel on every PR SO THAT I get a
  single combined quality verdict per PR.

## EARS specification
- `EARS-001` (Ubiquitous): THE SYSTEM SHALL run the spec-reviewer, coverage-gate, and architecture-guardian
  agents in parallel on every PR and synthesise one combined verdict.
- `EARS-002` (Event-driven): WHEN a PR is opened THE SYSTEM SHALL fan out the three agents concurrently and
  combine their results (max-severity wins).

## Acceptance criteria
1. Given a PR, When the orchestrator runs, Then all three agents run in parallel and a single combined
   verdict is produced.
2. Given any agent returning a blocking verdict, When synthesised, Then the combined verdict blocks.

## Human interface test plan
None — the orchestrator is a fan-out-then-synthesise agent. Proof: against fixtured agent verdicts, all
three run concurrently and the combined verdict is produced; any one blocking verdict makes the combined
verdict block (max-severity wins) — asserted on the synthesised verdict, the three agents mocked.

## Implementation notes
- Depends on the three agents (PLAN_0062/0063/0064). Mirrors the existing fan-out-then-synthesise pattern
  (e.g. `/foundry:pr-review`): concurrent fan-out, max-severity-wins synthesis.
- The agents are mocked in the orchestrator's tests (deterministic — no real sub-agent spawn, no flakiness).
