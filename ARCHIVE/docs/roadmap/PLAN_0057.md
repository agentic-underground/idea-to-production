# Add code-quality trend tracking + regression alerts

**Brief**: Write quality scores to `.claude/quality-history.json` after each run; a `/quality-trend`
command shows improvement over time; regression detection alerts when a file's score drops.

## User stories
- AS a maintainer I WANT quality scores tracked over time SO THAT I can see trends and get alerted when a
  file regresses.

## EARS specification
- `EARS-001` (Ubiquitous): THE SYSTEM SHALL persist each run's quality scores to
  `.claude/quality-history.json`.
- `EARS-002` (Event-driven): WHEN `/quality-trend` runs THE SYSTEM SHALL show the score trend over time;
  WHEN a file's score drops versus its history THE SYSTEM SHALL flag a regression.

## Acceptance criteria
1. Given two runs, When the second completes, Then `quality-history.json` has both and `/quality-trend`
   shows the trend.
2. Given a file whose score dropped, When analysed, Then a regression alert is raised.

## Human interface test plan
None — trend tracking is a data file + a report command. Proof: two fixture runs produce two history
records; `/quality-trend` reconstructs the trend; a dropped file score raises a regression alert (asserted
on the history file + command output; timestamps injected for determinism).

## Implementation notes
- Append per-run scores to `.claude/quality-history.json` (append-only); `/quality-trend` reads it;
  per-file regression comparison vs. prior history.
- Timestamps injected in tests for determinism (no wall-clock flakiness).
