## Shared infrastructure
- **The `code-quality` skill + its lens/scoring engine** (`plugins/foundry/skills/code-quality/`) is the
  one surface every PLAN extends — the generic lenses, the score, and the `/code-quality` entry. None of
  these PLANs re-creates the engine; each adds a lens or a workflow over it.
- **The changed-scope computation** — PLAN_0056 (pre-commit hook, changed *files*) and PLAN_0058 (PR/diff
  mode, changed *lines*) both derive a `git diff` scope; share that scoping helper rather than duplicating
  it. The score-history file PLAN_0057 writes (`.claude/quality-history.json`) is read by the trend
  command and by regression detection.

## Definition of done
- All six PLANs are `completed` (the engine rolls EPIC completion up from its PLANs).
- `.pipeline/verify` is GREEN for each landed PLAN; `scripts/verify-prereqs.sh` (incl. check P, and check
  L hooks smoke-exec for the pre-commit hook) stays GREEN.

## Execution contract (for the builder)
The engine builds one PLAN at a time in `## Plans` order, running each PLAN's `## Construction process`
(FOUNDRY scoped to the `(EPIC, PLAN)` pair). A failed PLAN leaves its row `engaged` and resumes next tick;
the EPIC lands only when all six PLANs are GREEN.
