**Brief**: An `architecture-guardian-agent` that continuously enforces the project's declared
architectural boundaries, so the architecture doesn't erode between deliberate reviews.

## User stories
- AS an architect I WANT boundary violations caught continuously SO THAT the architecture doesn't erode
  between deliberate reviews.

## EARS specification
- `EARS-001` (Ubiquitous): THE SYSTEM SHALL enforce the project's declared architectural boundaries on an
  ongoing basis.
- `EARS-002` (Event-driven): WHEN a change crosses a declared boundary (e.g. a layer importing inward-only
  code the wrong way) THE SYSTEM SHALL flag the violation.

## Acceptance criteria
1. Given a declared boundary, When a change violates it, Then the violation is flagged with the offending
   dependency.
2. Given a boundary-respecting change, When checked, Then it passes.

## Human interface test plan
None — the guardian is an analysis agent over the dependency graph. Proof: a fixture project with a
declared boundary flags an inward-only violation with the offending dependency; a respecting change passes
(asserted on the agent's findings).
