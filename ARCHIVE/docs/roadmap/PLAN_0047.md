**Brief**: The server-side enforcement that earns the name — a **workflow-template allowlist**
(`submit_prompt` takes a `template_id`, never a node graph: removes arbitrary-node execution + URL-loading
SSRF), **input validation** (params schema-checked: steps/seed bounded, prompt length-capped, model ∈
allowlist), and a **model allowlist** (only checkpoints physically in the bind-mounted `DIFFUSION_MODELS`
are selectable).

## User stories
- AS a security owner I WANT the server to reject anything but allowlisted templates/models with validated
  params SO THAT a caller cannot execute an arbitrary node graph or load an arbitrary URL/model.

## EARS specification
- `EARS-001` (Ubiquitous): THE SYSTEM SHALL accept only allowlisted `template_id`s and allowlisted models,
  with every param schema-validated within bounds.
- `EARS-002` (Unwanted): IF a request supplies a raw node graph, an out-of-bounds param, or a model not in
  `DIFFUSION_MODELS` THEN THE SYSTEM SHALL refuse it with a validation error, executing nothing.

## Acceptance criteria
1. Given a node-graph payload, When submitted, Then it is refused (only `template_id` is accepted).
2. Given an out-of-range step/seed, an over-long prompt, or an unlisted model, When submitted, Then each is
   refused; given valid inputs, Then it is accepted.

## Human interface test plan
None — enforcement is server-side logic. Proof: a node-graph payload, out-of-range params, and an unlisted
model are each refused with a validation error and nothing executes; valid inputs pass (asserted on the
tool responses against a stubbed ComfyUI).

## Implementation notes
- Allowlist loaders read the shipped templates + the bind-mounted model set; param schema per template.
  Refuse-and-execute-nothing is the abuse-path invariant. This removes arbitrary-node execution and
  URL-loading-node SSRF.
- Refines PLAN_0046's `server/`; ComfyUI stubbed in tests (deterministic).
